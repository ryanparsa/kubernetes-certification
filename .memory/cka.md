# CKA Memory

Labs are numbered sequentially under `cka/<N>/`; find the next number with `ls cka/`.
`cka/29/` is the canonical reference implementation — copy its structure for every new lab.
Verbatim killer.sh source lives in `cka/ref/a.md` (labs 1-17, Simulator A) and `cka/ref/b.md` (labs 18-34, Simulator B).
Cluster name convention: `cka-lab-<N>` — must match the `name` field in `kind-config.yaml`.
Each lab directory must contain: `readme.md` and `assets/{kind-config.yaml,up.sh,down.sh,fix.sh,check.sh}`.
`check.py` is required when checks involve OR conditions, JSON parsing, or cross-resource assertions.
`course/` holds manifests created while solving; it is git-ignored and generated by `up.sh`.
Multi-node kind configs are only needed when the task involves scheduling or node management.
