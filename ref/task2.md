  ### Part 1: kubeadm (most complex — configures and initializes entire cluster)                                                                                                                                                                                                                  
  
  #### 1.1 — `kubeadm init`                                                                                                                                                                                                                                                                       
                  
  What it does internally (step by step):                                                                                                                                                                                                                                                         
  1. Preflight checks (kernel modules, ports, container runtime reachable)
  2. Generates PKI under /etc/kubernetes/pki/                                                                                                                                                                                                                                                     
  3. Writes kubeconfig files to /etc/kubernetes/                                                                                                                                                                                                                                                  
  4. Writes static pod manifests to /etc/kubernetes/manifests/                                                                                                                                                                                                                                    
  5. Waits for API server to become healthy                                                                                                                                                                                                                                                       
  6. Bootstraps RBAC rules for system components                                                                                                                                                                                                                                                  
  7. Installs CoreDNS and kube-proxy                                                                                                                                                                                                                                                              
  8. Prints the `kubeadm join` command with bootstrap token                                                                                                                                                                                                                                       
                                                                                                                                                                                                                                                                                                  
  Key flags to cover:
    --config                    # path to ClusterConfiguration YAML (preferred over flags)                                                                                                                                                                                                        
    --pod-network-cidr          # must match CNI plugin (e.g. 10.244.0.0/16 for flannel)                                                                                                                                                                                                          
    --service-cidr              # default 10.96.0.0/12                                                                                                                                                                                                                                            
    --apiserver-advertise-address  # IP the API server listens on                                                                                                                                                                                                                                 
    --control-plane-endpoint    # HA load balancer DNS/IP                                                                                                                                                                                                                                         
    --upload-certs              # upload control-plane certs to cluster (for HA join)                                                                                                                                                                                                             
    --skip-phases               # skip specific init phases                                                                                                                                                                                                                                       
    --dry-run                   # simulate without writing files                                                                                                                                                                                                                                  
                                                                                                                                                                                                                                                                                                  
  ClusterConfiguration YAML structure (full annotated example)                                                                                                                                                                                                                                    
  
  Output to understand:                                                                                                                                                                                                                                                                           
    [preflight] ...
    [certs] ...                                                                                                                                                                                                                                                                                   
    [kubeconfig] ...
    [controlplane] ...
    [etcd] ...
    [wait-control-plane] ...                                                                                                                                                                                                                                                                      
    [apiclient] ...
    [uploadconfig] ...                                                                                                                                                                                                                                                                            
    [kubelet] ... 
    [upload-certs] ...                                                                                                                                                                                                                                                                            
    [mark-control-plane] ...
    [bootstrap-token] ...                                                                                                                                                                                                                                                                         
    [addons] ...  
                                                                                                                                                                                                                                                                                                  
  What each phase line means; which files are created at each step.                                                                                                                                                                                                                               
                                                                                                                                                                                                                                                                                                  
  #### 1.2 — `kubeadm join`                                                                                                                                                                                                                                                                       
                  
  Two modes:
  - Worker join: presents bootstrap token → receives kubelet client cert → joins cluster
  - Control plane join: `--control-plane --certificate-key` → downloads PKI → becomes master                                                                                                                                                                                                      
                                                                                                                                                                                                                                                                                                  
  Key flags:                                                                                                                                                                                                                                                                                      
    --token                     # bootstrap token (format: abcdef.0123456789abcdef)                                                                                                                                                                                                               
    --discovery-token-ca-cert-hash  # SHA256 of cluster CA (pins the CA)                                                                                                                                                                                                                          
    --control-plane             # join as control plane node                                                                                                                                                                                                                                      
    --certificate-key           # decrypts uploaded certs (HA setup)                                                                                                                                                                                                                              
    --apiserver-advertise-address  # override for this control plane node                                                                                                                                                                                                                         
                                                                                                                                                                                                                                                                                                  
  Internal flow for worker join:                                                                                                                                                                                                                                                                  
  1. Kubelet starts with bootstrap kubeconfig (uses token as credential)
  2. Submits CertificateSigningRequest to API server                                                                                                                                                                                                                                              
  3. Auto-approved by bootstrap RBAC
  4. Kubelet writes dated .pem and flips kubelet-client-current.pem symlink                                                                                                                                                                                                                       
  5. Kubelet now authenticates with real cert, stops using bootstrap token                                                                                                                                                                                                                        
                                                                                                                                                                                                                                                                                                  
  #### 1.3 — `kubeadm certs`                                                                                                                                                                                                                                                                      
                                                                                                                                                                                                                                                                                                  
  Subcommands:    
    kubeadm certs check-expiration        # shows each cert, days remaining, CA it was signed by
    kubeadm certs renew all               # renew all certs at once                                                                                                                                                                                                                               
    kubeadm certs renew <name>            # renew one specific cert                                                                                                                                                                                                                               
    kubeadm certs certificate-key         # generate a new certificate key for HA                                                                                                                                                                                                                 
    kubeadm certs generate-csr            # generate CSR files (for external CA signing)                                                                                                                                                                                                          
                                                                                                                                                                                                                                                                                                  
  Output of `check-expiration` — how to read it:                                                                                                                                                                                                                                                  
    CERTIFICATE                EXPIRES                  RESIDUAL TIME  EXTERNALLY MANAGED                                                                                                                                                                                                         
    admin.conf                 Apr 24, 2027 09:56 UTC   364d           no                                                                                                                                                                                                                         
    ...                                                                                                                                                                                                                                                                                           
                                                                                                                                                                                                                                                                                                  
  Fields: CERTIFICATE name, expiry date, days remaining, whether managed externally (cert-manager).                                                                                                                                                                                               
                  
  #### 1.4 — `kubeadm token`                                                                                                                                                                                                                                                                      
  
    kubeadm token create                  # create a new bootstrap token                                                                                                                                                                                                                          
    kubeadm token create --print-join-command  # full join command
    kubeadm token list                    # show all tokens + expiry                                                                                                                                                                                                                              
    kubeadm token delete <token>          # revoke a token                                                                                                                                                                                                                                        
                                                                                                                                                                                                                                                                                                  
  Bootstrap token format: `<6-char>.<16-char>` (e.g. `abcdef.0123456789abcdef`)                                                                                                                                                                                                                   
  Stored as Secrets in kube-system namespace: `bootstrap-token-<token-id>`
                                                                                                                                                                                                                                                                                                  
  #### 1.5 — `kubeadm upgrade`
                                                                                                                                                                                                                                                                                                  
    kubeadm upgrade plan                  # shows available versions, checks etcd, lists certs
    kubeadm upgrade apply v1.32.0         # upgrades control plane components
    kubeadm upgrade node                  # upgrades worker node kubelet config
                                                                                                                                                                                                                                                                                                  
  Internal steps of `upgrade apply`:
  1. Checks preflight (version skew policy, etcd compatibility)                                                                                                                                                                                                                                   
  2. Downloads new component images                                                                                                                                                                                                                                                               
  3. Updates static pod manifests (triggers kubelet to restart them)                                                                                                                                                                                                                              
  4. Renews all certificates that expire within 180 days                                                                                                                                                                                                                                          
  5. Updates kubelet config ConfigMap in kube-system                                                                                                                                                                                                                                              
  6. Updates RBAC rules for new version                                                                                                                                                                                                                                                           
  
  After upgrade: must manually `apt upgrade kubelet kubectl` on each node.                                                                                                                                                                                                                        
                  
  #### 1.6 — `kubeadm config`                                                                                                                                                                                                                                                                     
                  
    kubeadm config print init-defaults     # shows full default ClusterConfiguration YAML                                                                                                                                                                                                         
    kubeadm config print join-defaults     # shows JoinConfiguration defaults
    kubeadm config images list            # lists all images kubeadm will pull for this version                                                                                                                                                                                                   
    kubeadm config images pull            # pre-pulls all images (useful for air-gapped)                                                                                                                                                                                                          
    kubeadm config migrate               # migrates old config format to new API version                                                                                                                                                                                                          
                                                                                                                                                                                                                                                                                                  
  #### 1.7 — `kubeadm reset`                                                                                                                                                                                                                                                                      
                  
  Undoes everything kubeadm init/join did:                                                                                                                                                                                                                                                        
  - Stops static pods (removes manifests)
  - Resets iptables/ipvs rules                                                                                                                                                                                                                                                                    
  - Removes /etc/kubernetes/ directory
  - Resets kubelet state                                                                                                                                                                                                                                                                          
  - Does NOT remove etcd data (must do manually: rm -rf /var/lib/etcd)
  - Does NOT remove CNI state (must do manually: rm -rf /etc/cni /opt/cni)                                                                                                                                                                                                                        
                                                                                                                                                                                                                                                                                                  
  #### 1.8 — `kubeadm phase` (granular control)                                                                                                                                                                                                                                                   
                                                                                                                                                                                                                                                                                                  
  Allows running individual phases of init/join:                                                                                                                                                                                                                                                  
    kubeadm init phase certs all
    kubeadm init phase kubeconfig all                                                                                                                                                                                                                                                             
    kubeadm init phase control-plane all
    kubeadm init phase etcd local                                                                                                                                                                                                                                                                 
    kubeadm init phase upload-certs                                                                                                                                                                                                                                                               
  
  Useful for debugging exactly what step fails, or for custom cluster setups.

  ### Part 3: kubelet
                                                                                                                                                                                                                                                                                                  
  #### 3.1 — What kubelet Does

  - Watches API server (or static pod manifests) for pod specs assigned to this node
  - Calls CRI (containerd via /run/containerd/containerd.sock) to start/stop containers                                                                                                                                                                                                           
  - Reports pod status back to API server                                                                                                                                                                                                                                                         
  - Serves health endpoints on port 10248 (local) and 10250 (API server calls)                                                                                                                                                                                                                    
  - Manages volume mounts, secrets injection, service account token projection                                                                                                                                                                                                                    
  - Runs liveness/readiness/startup probes                                                                                                                                                                                                                                                        
  - Manages certificate rotation (client cert via CSR API)                                                                                                                                                                                                                                        
                                                                                                                                                                                                                                                                                                  
  #### 3.2 — Configuration                                                                                                                                                                                                                                                                        
                                                                                                                                                                                                                                                                                                  
  kubelet is configured two ways (flags are deprecated in favor of config file):
                                                                                                                                                                                                                                                                                                  
    /var/lib/kubelet/config.yaml          # KubeletConfiguration object
    /var/lib/kubelet/kubeadm-flags.env    # remaining CLI flags set by kubeadm                                                                                                                                                                                                                    
                                                                                                                                                                                                                                                                                                  
  Key KubeletConfiguration fields to know:                                                                                                                                                                                                                                                        
    clusterDNS: ["10.96.0.10"]           # CoreDNS cluster IP                                                                                                                                                                                                                                     
    clusterDomain: cluster.local                                                                                                                                                                                                                                                                  
    containerRuntimeEndpoint: unix:///run/containerd/containerd.sock
    staticPodPath: /etc/kubernetes/manifests                                                                                                                                                                                                                                                      
    authentication:
      anonymous: {enabled: false}         # must be false for security                                                                                                                                                                                                                            
      webhook: {enabled: true}                                                                                                                                                                                                                                                                    
    authorization:
      mode: Webhook                       # delegates to API server                                                                                                                                                                                                                               
    serverTLSBootstrap: true             # enables server cert auto-rotation
    rotateCertificates: true             # enables client cert rotation                                                                                                                                                                                                                           
    evictionHard:                                                                                                                                                                                                                                                                                 
      memory.available: "100Mi"                                                                                                                                                                                                                                                                   
      nodefs.available: "10%"                                                                                                                                                                                                                                                                     
    cpuManagerPolicy: static             # for CPU pinning (Guaranteed QoS pods)
                                                                                                                                                                                                                                                                                                  
  #### 3.3 — Key Endpoints
                                                                                                                                                                                                                                                                                                  
    :10250/healthz                        # kubelet health
    :10250/pods                           # list all pods on node (JSON)                                                                                                                                                                                                                          
    :10250/stats/summary                  # resource usage stats                                                                                                                                                                                                                                  
    :10250/logs/<logfile>                 # node log files                                                                                                                                                                                                                                        
    :10248/healthz                        # local-only health (no auth)                                                                                                                                                                                                                           
                                                                                                                                                                                                                                                                                                  
  #### 3.4 — Systemd Unit

  ### Part 4: kube-apiserver

  #### 4.1 — What It Does

  The front door to the cluster. All kubectl commands, controller reconciliation, scheduler
  decisions, and kubelet status reports go through the API server. It:                                                                                                                                                                                                                            
  - Authenticates every request (client cert, token, OIDC, etc.)                                                                                                                                                                                                                                  
  - Authorizes every request (RBAC, Node authorizer)                                                                                                                                                                                                                                              
  - Runs admission controllers (mutate then validate)                                                                                                                                                                                                                                             
  - Persists objects to etcd                                                                                                                                                                                                                                                                      
  - Serves watch streams to controllers and kubelets
                                                                                                                                                                                                                                                                                                  
  #### 4.2 — Run Mode
                                                                                                                                                                                                                                                                                                  
  Runs as a static pod. Manifest: `/etc/kubernetes/manifests/kube-apiserver.yaml`
                                                                                                                                                                                                                                                                                                  
  Editing the manifest triggers kubelet to automatically restart the API server.
                                                                                                                                                                                                                                                                                                  
  #### 4.3 — Critical Flags
                                                                                                                                                                                                                                                                                                  
  Authentication: 
    --client-ca-file=/etc/kubernetes/pki/ca.crt
    --tls-cert-file=/etc/kubernetes/pki/apiserver.crt                                                                                                                                                                                                                                             
    --tls-private-key-file=/etc/kubernetes/pki/apiserver.key
    --service-account-key-file=/etc/kubernetes/pki/sa.pub                                                                                                                                                                                                                                         
    --service-account-issuer=https://kubernetes.default.svc.cluster.local
    --anonymous-auth=false                # security: must be false                                                                                                                                                                                                                               
                  
  Authorization:                                                                                                                                                                                                                                                                                  
    --authorization-mode=Node,RBAC
                                                                                                                                                                                                                                                                                                  
  Etcd connection:
    --etcd-servers=https://127.0.0.1:2379                                                                                                                                                                                                                                                         
    --etcd-cafile=/etc/kubernetes/pki/etcd/ca.crt
    --etcd-certfile=/etc/kubernetes/pki/apiserver-etcd-client.crt                                                                                                                                                                                                                                 
    --etcd-keyfile=/etc/kubernetes/pki/apiserver-etcd-client.key                                                                                                                                                                                                                                  
                                                                                                                                                                                                                                                                                                  
  Admission:

  ### Part 5: kube-scheduler

  #### 5.1 — What It Does
                                                                                                                                                                                                                                                                                                  
  Watches for unscheduled pods (pods with no `.spec.nodeName`), runs the scheduling algorithm
  (filter → score → bind), then writes the chosen node name back to the pod spec via the API server.                                                                                                                                                                                              
                                                                                                                                                                                                                                                                                                  
  Never talks directly to kubelets or containerd — only to the API server.                                                                                                                                                                                                                        
                                                                                                                                                                                                                                                                                                  
  #### 5.2 — Run Mode
                                                                                                                                                                                                                                                                                                  
  Static pod: `/etc/kubernetes/manifests/kube-scheduler.yaml`
                                                                                                                                                                                                                                                                                                  
  #### 5.3 — Key Flags
                                                                                                                                                                                                                                                                                                  
    --config=/etc/kubernetes/scheduler.conf      # kubeconfig (auth to API server)
    --leader-elect=true                          # HA: only one scheduler active at a time                                                                                                                                                                                                        
    --profiling=false                            # CKS: disable                                                                                                                                                                                                                                   
    --config=<KubeSchedulerConfiguration file>  # scheduling profiles, plugins                                                                                                                                                                                                                    
                                                                                                                                                                                                                                                                                                  
  Scheduler identity: CN = `system:kube-scheduler`, O = none                                                                                                                                                                                                                                      
  Maps to the built-in `system:kube-scheduler` ClusterRole.                                                                                                                                                                                                                                       
                  
  ---                                                                                                                                                                                                                                                                                             
                  
  ### Part 6: kube-controller-manager
                                                                                                                                                                                                                                                                                                  
  #### 6.1 — What It Does
                                                                                                                                                                                                                                                                                                  
  Runs all built-in control loops (controllers). Each controller watches resources and reconciles
  actual state toward desired state. Key controllers:                                                                                                                                                                                                                                             
                                                                                                                                                                                                                                                                                                  
    Node controller         — marks nodes NotReady, evicts pods when node unreachable                                                                                                                                                                                                             
    Deployment controller   — creates/updates ReplicaSets                                                                                                                                                                                                                                         
    ReplicaSet controller   — creates/deletes pods to match desired replica count                                                                                                                                                                                                                 
    Job controller          — creates pods for batch Jobs, tracks completions                                                                                                                                                                                                                     
    ServiceAccount controller — creates default SA + token for new namespaces                                                                                                                                                                                                                     
    Namespace controller    — handles namespace deletion (finalizers)                                                                                                                                                                                                                             
    EndpointSlice controller — keeps EndpointSlices in sync with pod IPs                                                                                                                                                                                                                          
    CertificateSigningRequest controller — auto-approves kubelet bootstrap CSRs                                                                                                                                                                                                                   
    Token controller        — signs ServiceAccount JWT tokens using sa.key                                                                                                                                                                                                                        
                                                                                                                                                                                                                                                                                                  
  #### 6.2 — Run Mode                                                                                                                                                                                                                                                                             
                                                                                                                                                                                                                                                                                                  
  Static pod: `/etc/kubernetes/manifests/kube-controller-manager.yaml`

  #### 6.3 — Key Flags

    --kubeconfig=/etc/kubernetes/controller-manager.conf
    --service-account-private-key-file=/etc/kubernetes/pki/sa.key   # signs SA tokens                                                                                                                                                                                                             
    --root-ca-file=/etc/kubernetes/pki/ca.crt                                                                                                                                                                                                                                                     
    --cluster-signing-cert-file=/etc/kubernetes/pki/ca.crt          # signs kubelet certs                                                                                                                                                                                                         
    --cluster-signing-key-file=/etc/kubernetes/pki/ca.key                                                                                                                                                                                                                                         
    --leader-elect=true
    --profiling=false                                                                                                                                                                                                                                                                             
    --node-monitor-grace-period=40s      # time before node marked Unreachable
    --pod-eviction-timeout=5m0s          # time before pods evicted from Unreachable node                                                                                                                                                                                                         
    --controllers=*,bootstrapsigner,tokencleaner   # which controllers to run (* = all)                                                                                                                                                                                                           
                                                                                                                                                                                                                                                                                                  
  Controller-manager identity: CN = `system:kube-controller-manager`                                                                                                                                                                                                                              
  Maps to built-in `system:kube-controller-manager` ClusterRole.                                                                                                                                                                                                                                  
                  
  ---                                                                                                                                                                                                                                                                                             
                  
  ### Part 7: containerd Ecosystem (containerd, crictl, ctr)
                                                                                                                                                                                                                                                                                                  
  #### 7.1 — containerd
                                                                                                                                                                                                                                                                                                  
  The container runtime. Kubelet talks to it via the CRI gRPC socket:
    /run/containerd/containerd.sock                                                                                                                                                                                                                                                               
  
  Config: `/etc/containerd/config.toml`                                                                                                                                                                                                                                                           
  Key config sections:                                                                                                                                                                                                                                                                            
    [plugins."io.containerd.grpc.v1.cri"]
      sandbox_image = "registry.k8s.io/pause:3.9"                                                                                                                                                                                                                                                 
    [plugins."io.containerd.grpc.v1.cri".containerd.runtimes.runc]
      runtime_type = "io.containerd.runc.v2"                                                                                                                                                                                                                                                      
    [plugins."io.containerd.grpc.v1.cri".containerd.runtimes.runc.options]                                                                                                                                                                                                                        
      SystemdCgroup = true               # must match kubelet cgroupDriver                                                                                                                                                                                                                        
                                                                                                                                                                                                                                                                                                  
    systemctl status containerd
    systemctl restart containerd                                                                                                                                                                                                                                                                  
                                                                                                                                                                                                                                                                                                  
  #### 7.2 — crictl (CRI CLI — use this for node debugging)
                                                                                                                                                                                                                                                                                                  
  crictl talks directly to the CRI socket, bypassing kubelet. Essential for debugging pods that                                                                                                                                                                                                   
  won't start.                                                                                 
                                                                                                                                                                                                                                                                                                  
    crictl ps                            # running containers (not pods)                                                                                                                                                                                                                          
    crictl ps -a                         # all containers including stopped                                                                                                                                                                                                                       
    crictl pods                          # pod-level sandbox list                                                                                                                                                                                                                                 
    crictl images                        # cached images on node 
    crictl pull <image>                  # pull an image                                                                                                                                                                                                                                          
    crictl inspect <container-id>        # full container config + state JSON
    crictl logs <container-id>           # container logs                                                                                                                                                                                                                                         
    crictl exec -it <container-id> sh    # exec into container                                                                                                                                                                                                                                    
    crictl stats                         # CPU/memory per container                                                                                                                                                                                                                               
    crictl info                          # runtime info (version, containerd config)                                                                                                                                                                                                              
    crictl rm <container-id>             # remove container                                                                                                                                                                                                                                       
    crictl rmp <pod-id>                  # remove pod sandbox                                                                                                                                                                                                                                     
    crictl stopp <pod-id>                # stop pod sandbox                                                                                                                                                                                                                                       
                                                                                                                                                                                                                                                                                                  
  Config: `/etc/crictl.yaml`
    runtime-endpoint: unix:///run/containerd/containerd.sock                                                                                                                                                                                                                                      
    image-endpoint: unix:///run/containerd/containerd.sock                                                                                                                                                                                                                                        
                                                          
  #### 7.3 — ctr (low-level containerd CLI — for image management)                                                                                                                                                                                                                                
                                                                  
    ctr images list                      # all images in containerd store                                                                                                                                                                                                                         
    ctr images pull <image>                                              
    ctr containers list                  # all containers (not just k8s ones)                                                                                                                                                                                                                     
    ctr tasks list                       # running processes                 
    ctr namespaces list                  # containerd namespaces (k8s uses "k8s.io")                                                                                                                                                                                                              
    ctr -n k8s.io images list            # images in the kubernetes namespace                                                                                                                                                                                                                     
                                                                                                                                                                                                                                                                                                  
  #### 7.4 — etcdctl (bonus — essential for CKS)                                                                                                                                                                                                                                                  
                                                                                                                                                                                                                                                                                                  
    ETCDCTL_API=3 etcdctl \                                                                                                                                                                                                                                                                       
      --endpoints=https://127.0.0.1:2379 \
      --cacert=/etc/kubernetes/pki/etcd/ca.crt \                                                                                                                                                                                                                                                  
      --cert=/etc/kubernetes/pki/etcd/server.crt \
      --key=/etc/kubernetes/pki/etcd/server.key \                                                                                                                                                                                                                                                 
      <subcommand>                                                                                                                                                                                                                                                                                
                  
    etcdctl member list                  # shows all etcd cluster members + health                                                                                                                                                                                                                
    etcdctl endpoint health              # health of each endpoint                
    etcdctl endpoint status              # leader election status                                                                                                                                                                                                                                 
    etcdctl get /registry/secrets/default/mysecret   # raw read (verify encryption)
    etcdctl get / --prefix --keys-only   # list all keys in etcd                                                                                                                                                                                                                                  
    etcdctl snapshot save /tmp/etcd.db   # backup snapshot                                                                                                                                                                                                                                        
    etcdctl snapshot restore /tmp/etcd.db --data-dir=/var/lib/etcd-restore  # restore
